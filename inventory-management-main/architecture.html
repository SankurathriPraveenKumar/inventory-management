Data Flow
1

Server Startup — JSON loaded into memory

mock_data.py reads all server/data/*.json files once at startup. Data is held as Python lists/dicts in memory for the lifetime of the process.

2

User applies filters in the browser

FilterBar.vue binds four dropdowns to reactive refs in useFilters.js — Time Period, Location, Category, Order Status. Changing any filter triggers a watcher in the active view.

3

Vue view calls api.js with current filters

getCurrentFilters() assembles a params object. api.js converts it to URLSearchParams and fires an Axios GET request — e.g. GET /api/orders?warehouse=London&status=Delivered&month=2025-03

4

FastAPI filters in-memory data

apply_filters() sequentially filters by warehouse, category, and status (skips any param set to 'all'). filter_by_month() then filters by direct month string or expands a quarter like Q1-2025 into three month ranges.

5

Pydantic validates and serializes response

Each endpoint declares a response_model (e.g. List[Order]). Pydantic validates the filtered list and serializes it to JSON before returning.

6

Vue stores response in reactive ref, template re-renders

The view stores the array in a ref([]). Computed properties derive KPIs and chart data from it. The template re-renders via Vue's reactivity system with no manual DOM work.

7

User sees updated tables, KPIs, and charts

All views (Dashboard, Orders, Inventory, Spending, etc.) share the same filter state via the singleton useFilters composable, so switching tabs preserves the active filter context.

Filter System
Time Period
param: month
all (default) 2025-01 … 2025-12 Q1-2025 … Q4-2025
Location
param: warehouse
all (default) San Francisco London Tokyo
Category
param: category
all (default) Circuit Boards Sensors Actuators Controllers Power Supplies
Order Status
param: status
all (default) Delivered Shipped Processing Backordered

Note: /api/inventory only supports warehouse + category. /api/backlog and /api/demand accept no filters. All filter logic runs server-side.
